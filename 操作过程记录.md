## 1. 项目概述

用电子化方案替代车间纸质巡检登记表。巡检员手机浏览器扫码即可填表提交，管理者可按设备、日期、人员查询历史记录。架构采用单文件 HTML 前端 + Express/SQLite 后端。

## 3. 架构决策记录

| 决策 | 选择 | 原因 | 拒绝方案 |
|------|------|------|----------|
| 存储引擎 | SQLite WAL | 并发写入安全，单文件部署 | `fs.appendFile`（并发损坏JSON） |
| 时间戳来源 | 服务器端 | 避免手机时钟不一致 | 客户端时间戳 |
| 认证 | 不做 | 内网信任边界足够，IP+时间戳审计 | 登录系统（延后到P3） |
| CORS | 显式 origin | 防止任意来源伪造请求 | 通配符 `*` |
| 设备编号 | 静态 dropdown | 防止 typo 产生 phantom 设备 | 自由文本输入 |
| 离线重试 | IndexedDB retry 队列 | 网络瞬断不丢数据 | 无重试 |
| 部署方式 | git clone + npm install + node | 最简部署 | 容器化/CI/CD |

## 4. 部署方式

```bash
git clone https://github.com/cloudzun/equipment-inspection-form.git
cd equipment-inspection-form
npm install
node server.js
```

每台设备贴二维码，URL 格式：`http://服务器IP:3000/?device=设备编号`
